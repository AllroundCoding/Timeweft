> For the judgement-call policy (the two-zone rule, determinism), see [`conventions.md`](conventions.md).

## Conventions that bite at these versions

### Artisan commands — attribute signatures (Laravel 13)

Laravel 13 commands declare their signature with PHP attributes, not `protected $signature`
properties:

```php
use Illuminate\Console\Attributes\Description;
use Illuminate\Console\Attributes\Signature;
use Illuminate\Console\Command;

#[Signature('docs:check-stack {--check : Fail instead of writing if the table is stale}')]
#[Description('Regenerate the version table in the stack cheatsheet from composer.lock')]
class CheckStack extends Command
{
    public function handle(): int  // type-hinted deps are injected by the container
    {
        return self::SUCCESS;       // self::SUCCESS / self::FAILURE / self::INVALID
    }
}
```

Commands in `app/Console/Commands` are auto-registered — there is no `Kernel::$commands` array to
update. Scheduling and routing live in `bootstrap/app.php` and `routes/console.php`, not an
`app/Console/Kernel.php`.

### Configuration & environment (Laravel 13)

- Read settings with `config('services.linear.key')`. **Never call `env()` outside `config/`** — once
  config is cached, `env()` returns `null`.
- The framework bootstraps from `bootstrap/app.php`.

### Testing — PHPUnit 12 (not Pest)

The suite is PHPUnit 12, which is **attribute-driven** — annotations like `/** @test */` and
`@dataProvider` are removed. Use attributes:

```php
use PHPUnit\Framework\Attributes\Test;
use PHPUnit\Framework\Attributes\DataProvider;

#[Test]
public function it_reproduces_the_same_chronicle_for_a_seed(): void { /* … */ }

#[DataProvider('seeds')]
public function test_seeds(int $seed): void { /* … */ }
```

Create tests with `php artisan make:test --phpunit {name}` (add `--unit` for unit tests). Run a
focused subset: `php artisan test --compact --filter=CheckStack`.

### PHP 8.4 idioms

- **`new` without parentheses to chain:** `new Rng($seed)->int(1, 6)`.
- **`Random\Randomizer` + a seeded engine** (`Random\Engine\Mt19937`) is the modern, deterministic RNG
  — exactly what [`App\Sim\Support\Rng`](../app/Sim/Support/Rng.php) wraps. Don't reach for the legacy
  `mt_srand()` / `mt_rand()` globals.
- **Property hooks** and **asymmetric visibility** (`public private(set)`) exist in 8.4; prefer them
  over hand-written getters at the boundary. The deterministic core still favours plain `readonly`
  value objects.
- Array helpers `array_find`, `array_any`, `array_all` are available.

### Pint 1.x

- Default preset is `laravel`. Config lives in `pint.json` (`preset`, `rules`, `exclude`, `notPath`).
- `vendor/bin/pint` fixes; `vendor/bin/pint --test` checks without writing (CI). Use
  `vendor/bin/pint --dirty` to format only changed files before finalizing.

### Tailwind CSS 4

- **CSS-first config.** There is no `tailwind.config.js` by default — import with `@import "tailwindcss";`
  and declare design tokens with the `@theme { … }` directive in your CSS.
- Container queries are built in (`@container`, `@sm:`, `@max-md:` …).

### Carbon 3 (boundary only)

Carbon 3 is available for boundary date handling, but the **simulation core uses ticks, never
Carbon** — see the no-wall-clock rule in [`conventions.md`](conventions.md#1-determinism--same-seed--same-chronicle).

## Keeping this current

The version table is regenerated by an Artisan command, mirroring the existing
`docs:pull-linear` / `docs:push-linear` design-doc sync:

```bash
php artisan docs:check-stack          # rewrite the table between the stack:versions markers
php artisan docs:check-stack --check  # CI mode: exit non-zero if the committed table is stale
```

The [`sync-stack-versions`](../.github/workflows/sync-stack-versions.yml) workflow runs this on a
schedule and whenever `composer.lock` changes, opening a PR if the table drifted. When a dependency
makes a **major** jump, refresh the prose above too — re-run Boost's `search-docs` locally for the
affected package and fold the gotchas back into this file.
